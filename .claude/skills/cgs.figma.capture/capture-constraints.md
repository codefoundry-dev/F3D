# Figma HTML-to-Design Capture Constraints

Reference document for the capture skill. Read this before generating or modifying the
`figmaCapture()` helper in any capture script.

## Source of Truth

`tests/e2e/{feature-id}/capture-screens.spec.ts` — the `figmaCapture()` function generated by this
skill.

## CSS Variable Resolution

Figma's HTML-to-Design converter does not evaluate CSS custom properties (`var(--name)`). All
`var()` references must be resolved to literal values before capture.

### In `<style>` Tags

Bundlers (Vite, webpack, etc.) may inject CSS modules as `<style>` elements. Replace `var(--name)`
with the computed value from `:root`:

```typescript
await page.evaluate(() => {
  const rootStyles = getComputedStyle(document.documentElement);
  document.querySelectorAll('style').forEach((styleEl) => {
    styleEl.textContent = styleEl.textContent!.replace(
      /var\(--([^)]+)\)/g,
      (_match, varName) => rootStyles.getPropertyValue('--' + varName).trim() || _match,
    );
  });
});
```

### In Inline Styles

Iterate all elements and replace inline `var()` references with their computed values:

```typescript
await page.evaluate(() => {
  document.querySelectorAll('*').forEach((el) => {
    const computed = getComputedStyle(el);
    const inline = (el as HTMLElement).style;
    for (let i = 0; i < inline.length; i++) {
      const prop = inline[i];
      if (inline.getPropertyValue(prop).includes('var(')) {
        inline.setProperty(prop, computed.getPropertyValue(prop));
      }
    }
  });
});
```

## Viewport Dimension Stamping

Figma's converter needs explicit pixel dimensions on root containers. Without them, layouts collapse
to narrow widths or incorrect heights.

Stamp **both width and height** on `<html>` and `<body>`:

```typescript
await page.evaluate(() => {
  const vw = window.innerWidth + 'px';
  const vh = window.innerHeight + 'px';
  document.documentElement.style.width = vw;
  document.documentElement.style.maxWidth = vw;
  document.documentElement.style.height = vh;
  document.documentElement.style.maxHeight = vh;
  document.documentElement.style.overflow = 'hidden';
  document.body.style.width = vw;
  document.body.style.maxWidth = vw;
  document.body.style.minWidth = vw;
  document.body.style.height = vh;
  document.body.style.maxHeight = vh;
  document.body.style.overflow = 'hidden';
});
```

**Critical**: Stamping only width causes height overflow issues where Figma frames render taller
than the viewport. Always stamp both dimensions.

## Flex/Grid Container Dimension Inlining

Elements using `display: flex`, `display: grid`, or `max-width` lose their computed dimensions in
Figma's converter. Inline **both width and height**:

```typescript
await page.evaluate(() => {
  const vw = window.innerWidth + 'px';
  const fullH = Math.max(document.documentElement.scrollHeight, window.innerHeight) + 'px';
  document.querySelectorAll('*').forEach((el) => {
    const computed = getComputedStyle(el);
    const htmlEl = el as HTMLElement;
    const display = computed.display;
    if (computed.position === 'fixed') {
      htmlEl.style.width = vw;
      htmlEl.style.height = fullH;
      htmlEl.style.position = 'absolute';
      htmlEl.style.left = '0';
      htmlEl.style.top = '0';
    } else if (display === 'flex' || display === 'grid' || computed.maxWidth !== 'none') {
      htmlEl.style.width = computed.width;
      htmlEl.style.height = computed.height;
    }
  });
});
```

**Critical**: Inlining only width on flex/grid containers causes layout distortion. Always inline
both dimensions.

## `position:fixed` to `position:absolute` Conversion

Figma's converter does not support `position: fixed`. Fixed-position elements (modals, overlays,
sticky headers) must be converted to `position: absolute` with explicit dimensions:

- Set `width` to viewport width
- Set `height` to full document scroll height (or viewport height, whichever is larger)
- Set `position: absolute`, `left: 0`, `top: 0`

This is handled in the flex/grid inlining step above — fixed elements are detected first and
converted before the flex/grid branch.

## `capture.js` Injection Protocol

The Figma capture script must be fetched and injected via Playwright, not loaded as a
`<script src>`:

```typescript
const response = await page
  .context()
  .request.get('https://mcp.figma.com/mcp/html-to-design/capture.js');
const scriptText = await response.text();
await page.evaluate((s) => {
  const el = document.createElement('script');
  el.textContent = s;
  document.head.appendChild(el);
}, scriptText);
await page.waitForTimeout(1000); // Wait for script initialization
```

### Timing

1. Inject script -> wait 1s for initialization
2. Call `window.figma.captureForDesign()` -> wait 10s for submission
3. Total per-capture overhead: ~11s

## `captureId` Lifecycle

1. **Generate**: Call `generate_figma_design` MCP tool with `outputMode: "existingFile"` and
   `fileKey` -> returns a `captureId` (UUID)
2. **Capture**: Pass `captureId` and endpoint to `window.figma.captureForDesign()`
3. **Poll**: After Playwright run, call `generate_figma_design` with just `captureId` every 5s up to
   10 times until status is `completed`

Endpoint format: `https://mcp.figma.com/mcp/capture/{captureId}/submit`

## `document.title` for Frame Naming

Set `document.title` before capture to control the Figma frame name:

```typescript
await page.evaluate((name) => {
  document.title = name;
}, frameName);
```

Format: `"{Screen Name} - {Viewport}"` (e.g., "Empty State - Desktop")

## Known Gotchas

| Issue                  | Cause                                       | Fix                                                |
| ---------------------- | ------------------------------------------- | -------------------------------------------------- |
| Height overflow        | Missing height stamp on `<html>`/`<body>`   | Set `height`, `maxHeight`, `overflow: hidden`      |
| CSS modules unresolved | Bundler injects `<style>` with `var()` refs | Process all `<style>` tags for variable resolution |
| 30s timeout            | Figma wait exceeds Playwright default       | Set `test.describe.configure({ timeout: 60_000 })` |
| Fire-and-forget        | `captureForDesign()` has no callback        | 10s wait is best-effort; no promise to await       |
| Fixed elements missing | `position: fixed` unsupported               | Convert to `position: absolute` with explicit dims |

## Viewport Dimensions Reference

| Viewport | Shorthand | Width | Height |
| -------- | --------- | ----- | ------ |
| Desktop  | D         | 1440  | 900    |
| Tablet   | T         | 768   | 1024   |
| Mobile   | M         | 375   | 812    |

## Execution Order Within `figmaCapture()`

1. Set `document.title` (frame naming)
2. Resolve CSS variables in `<style>` tags
3. Resolve CSS variables in inline styles
4. Stamp viewport dimensions on `<html>` and `<body>` (width + height)
5. Inline dimensions on flex/grid containers + convert `position:fixed`
6. Inject `capture.js` and trigger `captureForDesign()`
