Tests that fail during the login OTP step before reaching the actual test:

| #   | Test                               | User Story | Error                                                                    |
| --- | ---------------------------------- | ---------- | ------------------------------------------------------------------------ |
| 1   | AC1: View company profile          | US-1.09    | `OTP retry failed. Error: "Invalid code. Please try again." OTP: 505391` |
| 2   | AC1: View project list             | US-2       | `OTP retry failed. Error: "Invalid code. Please try again." OTP: 317366` |
| 3   | AC1: Upload invoice button visible | US-10.03   | `page.waitForURL: Timeout 15000ms — waiting for "**/verify-otp"`         |
| 4   | AC1: PO list page renders          | US-10.05   | `page.waitForURL: Timeout 15000ms — waiting for "**/verify-otp"`         |
| 5   | AC1: Invoice list page renders     | US-10.07   | `page.waitForURL: Timeout 15000ms — waiting for "**/verify-otp"`         |

**Root cause**: OTP verification is intermittently failing. Some tests (US-1.09, US-2) get past the
OTP page but receive "Invalid code"; others (US-10.03, 10.05, 10.07) time out waiting for the OTP
redirect. This may be a TOTP clock-drift issue or race condition in the test auth helper.

### Category 2: Strict Mode / Selector Issues (3 failures)

| #   | Test                              | User Story            | Error                                                          |
| --- | --------------------------------- | --------------------- | -------------------------------------------------------------- |
| 1   | AC3: Platform State table renders | US-12 (super-admin)   | `getByText('Component') resolved to 2 elements`                |
| 2   | AC3: Platform State table renders | US-12 (company-admin) | `getByText('Component') resolved to 2 elements`                |
| 3   | AC1: "Create new" + RFQ table     | US-10.04              | `locator('thead').getByText('Pick-up') resolved to 2 elements` |

**Root cause**: Ambiguous text selectors matching multiple DOM elements. The page has:

- "Component" appears in both a description paragraph and a table header
- "Pick-up" appears as a column header in two separate tables on the same page

**Fix**: Use more specific locators (e.g., `getByRole('columnheader', { name: 'Component' })` or
scope to a specific table).

## Recommendations

### Priority 1: Fix OTP auth helper (5 failures)

The OTP verification in the e2e login helper is intermittently failing with "Invalid code". This
blocks US-1.09, US-2, US-10.03, US-10.05, and US-10.07 from executing. Investigate:

- TOTP clock drift between test and backend
- Race condition in OTP generation/verification timing
- Whether the test seed user's OTP secret is stable across runs

### Priority 2: Fix ambiguous selectors (3 failures)

| Test                                 | Fix                                                                                   |
| ------------------------------------ | ------------------------------------------------------------------------------------- |
| US-12 AC3: `getByText('Component')`  | Use `getByRole('columnheader', { name: 'Component' })` or scope to `locator('table')` |
| US-10.04 AC1: `getByText('Pick-up')` | Use `.first()` or scope to a specific table with `locator('.rfq-table thead')`        |
